# 訂單卡片元件

訂單卡片系列：`OrderItem` / `OrderItemWithOK` / `ServedItem` / `AllItemDialog`。

## OrderItem（`widget/OrderItem.kt`）

主訂單卡片，用於 [[MainPage]] / [[ServedPage]] / [[RecallPage]] 三頁。

### 背景色（依 status）

| status | 顏色 |
|--------|------|
| `O` (PREPARED) | `ColorYellowBg` 黃 |
| `F` (COLLECTED) | `ColorPinkBg` 粉 |
| 其他 (J/W) | `ColorWhiteBg` 灰白 |

> 視覺一眼看出訂單階段：黃=可取餐、粉=已取餐。

### 結構

```
Card (15dp 圓角)
├─ 訂單號（縮寫顯示：前3碼+後5碼，用 - 連接）
├─ 訂單時間（只顯示時分秒部分）
├─ ─── Divider ───
├─ 品項清單（filter gType != "S"）
│   └─ OneLineItemInfo：${qty} x ${eName}#${flavor}#${addition}
│       └─ middleDetail（M/G 下接的 S 品項）內縮顯示
└─ featureBtn(detailSize) ← 由各 Page 自帶按鈕傳入
```

### middleDetail 組裝（M/G/S 處理）

`OrderItem.kt:68-79` 在渲染前迭代 detail：

```kotlin
for (i in 0 until orderDetail.size) {
    if (orderDetail[i].gType == "M" || orderDetail[i].gType == "G") {
        orderDetail[i].middleDetail = mutableListOf()
        for (j in i+1 until orderDetail.size) {
            if (orderDetail[j].gType == "S") {
                orderDetail[i].middleDetail = orderDetail[i].middleDetail?.plus(orderDetail[j])
            } else { break }
        }
    }
}
```

> ⚠️ **副作用：原地修改 `Order.detail`** — 違反 immutability。每次 OrderItem 重組都會跑這段，重複組裝同樣資料但因為 break 在第一個非 S 處停所以行為穩定。

### 顯示語系

```kotlin
val name = order.detail[i].eName        // 目前固定 eName
// 但 MainPage 的迭代用的是 prefs.itemDisplayLan 切換
```

> 不一致：OrderItem 內固定 eName，但其他地方（ServedItem、AllItemDialog）會看 prefs.itemDisplayLan。

## OrderItemWithOK（`widget/OrderItemWithOK.kt`）

「操作成功」覆蓋層。所有 Page 都用：

```kotlin
AnimatedVisibility(visible = !order.isVisible, enter = fadeIn(), exit = fadeOut()) {
    OrderItemWithOK(...)
}
```

內容：半透明背景（同 status bgColor 0.8 alpha）+ Lottie 動畫 `operation_success.json`。

> ServedPage 的 Collect 路徑沒設 `isVisible=false`，這個動畫不會觸發 — [[ISSUE-Collect動畫缺失]]。

## AllItemDialog（`widget/AllItemDialog.kt`）

[[MainPage]] 點卡片時的全項彈窗。寬 80% / 高 70%：
- 標題：訂單號（完整版，非縮寫）
- LazyColumn 列出所有 detail（含 flavor）
- 唯一按鈕：Finish（=派發 FinishOrder）

## 共通模式

所有訂單卡片：
- `featureBtn: @Composable (Int) -> Unit` 是傳進來的 slot，各 Page 提供不同按鈕（Main 用三態切換、Served 用 Reprint+Collect、Recall 用單 Recall）
- 卡片本身只負責「顯示」，不直接派發事件
- `Box` 把 OrderItem 與 OrderItemWithOK 疊起來，用 isVisible 切換覆蓋層

## 2026-06-03 更新：訂單號旁顯示震動機號碼

`OrderItem` 在訂單號後面加顯示震動機號碼（`order.note`）：

```kotlin
val pager = if (order.note.isNullOrBlank()) "" else " (${order.note})"
Text(text = stringResource(R.string.no) + shortNo + pager, ...)
```

- 來源：`Order.note`（`@Json(name="Note")`，見 [[資料模型]]），後端 Compass_KDS Orders 表的 `Note` 欄位。
- 空值不顯示括號。例：`NO. W01-00001 (1)`。
