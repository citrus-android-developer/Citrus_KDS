# 輪詢架構

App 透過兩個互斥的輪詢 Job 持續從 POS 拉取資料：

| Job | 啟動者 | 抓什麼 | 間隔 |
|-----|--------|--------|------|
| `orderInfoJob` | `startFetchKdsInfo` | 依 `currentPage` 抓 main/served/recall | 3 秒（動作後 1 秒） |
| `orderReadyJob` | `startFetchOrderReadyInfo` | `/controller/OrdersList` | 3 秒 |

兩者**互斥**：啟動其中一個必先 `stop` 另一個（`CentralViewModel.kt:503-523`）。

## fetchOrdersJob 詳細

`CentralViewModel.kt:772-784`：

```kotlin
private fun fetchOrdersJob(): Flow<Job> = flow {
    while (currentCoroutineContext().isActive) {
        if (!animateBufferGap) {
            fetchOrders()
            delay(3000)
        } else {
            delay(1000)
            animateBufferGap = false
            fetchOrders()
        }
    }
}
```

- 正常情況：每 3 秒輪一次
- 設了 `animateBufferGap=true`（執行完 SetOrderStatus 後會設）：等 1 秒再 fetch，配合 UI 卡片淡出動畫

## fetchOrders 只抓當前 Tab

`CentralViewModel.kt:822-866`：

```kotlin
repository.getOrders(type = currentState.currentPage.lowercase())
```

Response 依 `currentPage` 寫入不同 state 欄位（main/served/recall）。
切 Tab 時 `updateCurrentPage(index)` 改 `currentPage`，下次輪詢就拉新 list。

**副作用**：背景 Tab 的 list 不會即時刷新，需要靠：
- 操作當下對「非當前 Tab」list 做 `isVisible=false` 客戶端隱藏
- 切過去時的下一次輪詢真正同步

## fetchOrderReadyJob

`CentralViewModel.kt:786-791`：純粹固定 3 秒輪詢，沒有 animateBufferGap 機制（OrderReady 模式不會有 UI 操作）。

## 啟動點

| 進入畫面 | LaunchedEffect 觸發 | 對應 Job |
|----------|---------------------|----------|
| KdsScreen | `startFetchKdsInfo` | `orderInfoJob` |
| OrderReadyScreen | `startFetchOrderReadyInfo` | `orderReadyJob` |
