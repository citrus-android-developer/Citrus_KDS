# 資料層錯誤處理

從 API 呼叫到 UI 顯示錯誤訊息的完整鏈路。

## 三層型別

### 1. 後端回應包裝（`commonData/vo/ApiResult.kt`）

```kotlin
data class ApiResult<T>(
    @Json(name = "ApiStatus") val status: String,        // "1" = success
    @Json(name = "Data")      val data: T? = null,
    @Json(name = "Error")     val error: CodeMessage? = null,
)
data class CodeMessage(val code: String?, val message: String?)
```

POS 端統一回 `{ApiStatus, Data, Error}` 信封。`ApiStatus == "1"` 才算成功，其他都視為失敗。

### 2. 結果包裝（`commonData/Result.kt`）

```kotlin
typealias RootError = Error                  // commonData/Error.kt: sealed interface Error

sealed interface Result<out D, out E : RootError> {
    data class Success<D, E>(val data: D)
    data class Error<D, E>(val error: E)
    data class Loading<D, E>(val isLoading: Boolean)
}
```

三態 sealed interface，所有 Repository 方法回 `Flow<Result<T, RootError>>`。

### 3. 網路錯誤分類（`commonData/DataError.kt`）

```kotlin
sealed class NetworkError : DataError {
    data class UnknownError(errMsg)
    data class ServerError(errMsg)
    data class RequestTimeout(errMsg)
    data class HttpError(code, errMsg)
    data class NoInternet(errMsg)
    data class ConnectionError(errMsg)
    data class Serialization(errMsg)
    data class DataFetchFailed(errMsg)        // ApiStatus != "1" 時用這個
}
```

## resultFlowData（`util/FlowExt.kt`）

通用 wrapper，所有 Repository 方法經此封裝：

```
apiAction()
  ApiResult.status == "1" ? Result.Success(data) : Result.Error(DataFetchFailed)
        │
        ▼
.retryWithPolicy()
  IOException + attempt<2 → 延遲後重試（3s / 9s / 15s）
  其他 → emit RequestTimeout
        │
        ▼
.onIfLoadingAndCatch()
  onStart: emit Loading(true)
  catch:
    UnknownHostException → NoInternet
    HttpException 408    → RequestTimeout
    HttpException other  → HttpError(code)
    JsonParseException   → Serialization
    ConnectException     → ConnectionError
    else                 → UnknownError
        │
        ▼
.flowOn(Dispatchers.IO)
```

> 注意：retry 只重試 2 次，間隔 3s/9s。第 3 次直接報 RequestTimeout。

## UiText（語系安全的字串容器）

`util/UiText.kt`：

```kotlin
sealed class UiText {
    data class DynamicString(value)
    class StringResource(@StringRes id, arg)
    data class MultiUiText(uiTextList)        // 用於拼接多段
}
```

可 `asString()`（Compose）或 `asString(context)`（非 Compose），延遲到顯示時才解析 string resource，方便動態切換語系。

## asUiText 對應表

`util/asUiText.kt`：

```kotlin
NetworkError.X.asUiText() → MultiUiText(
    [DynamicString(errMsg), StringResource(對應的 R.string.X_error)]
)
```

把 NetworkError 轉成 UI 可顯示的 UiText。

## 完整鏈路範例

```
ViewModel.setOrderStatus
  → repository.setOrderStatus(request)
  → resultFlowData{ apiService.setOrderStatus(url, request) }
  → Flow<Result<Int, RootError>>
  → .collect:
      Result.Success → 更新 UI
      Result.Error(NetworkError) → setState { copy(errMsg = error.asUiText()) }
        ↓
ViewModel state.errMsg 改變
        ↓
KdsScreen LaunchedEffect(errMsg) → errShowing = true
        ↓
ErrorDialog(errMsg = state.errMsg) → 顯示中文 / 英文錯誤訊息（取決於 prefs.language）
```

## 重點

- 統一的錯誤模型，所有 API 共用
- 自動 retry（網路抖動）+ 自動 loading 狀態
- UiText 讓錯誤訊息隨語系切換
- 但 **errMsg 不會自動清除**，靠 UI 端 `onDismissErrorDialog` event（`CentralViewModel.kt` 應有 handler 但 grep 後沒看到實作 — 待查）
