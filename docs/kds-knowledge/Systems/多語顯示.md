# 多語顯示（i18n）

兩個獨立語言設定，務必分清：

| 設定 | 欄位 | 作用 |
|------|------|------|
| 系統語言 | `prefs.language`（English/华文） | 靜態 UI 字串（按鈕、頁面標題）→ `stringResource` |
| 訂單品項語言 | `prefs.itemDisplayLan`（English/华文/**English & 华文**） | 品項名稱、加料、調味（畫面＋列印），支援中英並列 |

## 套用機制
- **靜態字串**：[[MainActivity入口]] 以 `CompositionLocalProvider` 提供更新後 LocalConfiguration/LocalContext（languageState 單一來源），整棵樹一致重組（取代舊 updateConfiguration）。詳見 [[ISSUE-語系切換套用不一致]]。
- **品項顯示**：`Detail.nameDisplay(lan)`/`flavorDisplay(lan)`/`additionDisplay(lan)` 一律傳 `itemDisplayLan`。
  - 修正前畫面誤用 `prefs.language`（系統語言永不為『English & 华文』）→ 雙語永遠不出現；改用 itemDisplayLan 後畫面雙語生效。
  - 雙語格式：畫面『中 / 英』；列印『英 [中]』（格式略不同，皆雙語）。

## 字串資源（2026-06-05 新增）
values/ 與 values-zh-rSG/：on_sale/sold_out/set_sold_out/set_on_sale/qty/wastage/wastage_submitted/test_connection（reprint 沿用既有）。
庫存卡、損耗 Toast、測試連線、列印失敗重印 等按鈕全改 stringResource。

## 注意
- 印表機列印是 ESC/POS bytes，標題（如加點『加点/ADD ORDER』）依 itemDisplayLan 自行組字，非 Android 字串資源。


## 2026-06-08 更新：錯誤訊息全面本地化
所有使用者可見錯誤訊息一律走 `R.string` / `UiText.StringResource`，依語系切換（不再寫死中文）：
- 網路錯誤：`NetworkError` 全分支經 `asUiText()` 對應 string resource，新增 `ServerUrlNotSet`→`server_url_not_set`。
- 印表機測試：`printer_ip_required` / `printer_connect_success` / `printer_connect_failed`（後兩者帶 %1$s=ip:port）。
- APK 下載：`download_failed` 前綴 + 技術細節(MultiUiText)。
- 已知保留：網路錯誤的 feature 英文前綴(如 SetSellStatus:)為除錯識別標籤，不隨語系變。
相關：[[POS-API端點]] guard 的 ServerUrlNotSet、[[設定頁]]。
