# 列印系統

EPSON ePos 印表機驅動 + Compose UI 整合。三個檔案組成：

| 檔案 | 角色 |
|------|------|
| `util/PrintUtil.kt` | EPSON ePos SDK 包裝，負責 discover/connect/print/disconnect |
| `util/PrinterDetecter.kt` | Flow 中介者（單例），讓 ViewModel 觀察印表機狀態與發現結果 |
| `MainActivity.kt:145-149` | 觀察 `printOrder` state → 觸發 `printUtil.setOrderPrint(order)` |

## PrinterDetecter（單例 Flow Hub）

`PrinterDetecter.kt`：包含三個 `MutableSharedFlow`：

| Flow | 型別 | 用途 |
|------|------|------|
| `scannerValue` | `ArrayList<Map<String,String>>` | 探索到的印表機清單（PrinterName + Target） |
| `resetNotice` | `Unit` | 通知清空印表機清單 |
| `printStatus` | `PrintStatus` | 列印狀態（Idle/Printing/Error） |

ViewModel 在 init 區塊 `collectLatest` 這三個 flow，灌入 `state.printerInfo` / `state.printStatus`（`CentralViewModel.kt:73-95`）。

## PrintStatus sealed class

```kotlin
sealed class PrintStatus {
    object Idle : PrintStatus()
    object Printing : PrintStatus()
    data class Error(val errMsg: String) : PrintStatus()
}
```

## PrintUtil 初始化

`PrintUtil.kt` 在 `init` 啟動 `Discovery.start(context, filterOption, mDiscoveryListener)`：

- `FilterOption.deviceType = TYPE_PRINTER`
- `usbDeviceName = TRUE`
- 每發現一台印表機 → `mDiscoveryListener` 加入 `mPrinterList` → `printerDetecter.setValue(...)` 通知

UI 在 [[設定頁]] dropdown 顯示這個清單，使用者選一台 → `OnPrinterSelected` event 寫入 `prefs.printerName` + `prefs.printerTarget`。

## 接單即印廚房單（2026-06-04 起，printMode==0）

**接單(J→W)時自動印廚房單**（不再需要手動 Reprint 才有單）。兩條接單路徑都會觸發：
- **手動接單**：MainPage 按鈕 → `AcceptOrder` → `if (prefs.printMode == 0) requestPrint(toPrint)`（`CentralViewModel.kt:444-450`）
- **自動接單成功**：autoAccept 派發成功 → `if (prefs.printMode == 0) requestPrint(...)`（`CentralViewModel.kt:1185-1191`，見 [[自動接單功能]]）

`requestPrint(order)` = `setState { copy(printOrder = order, printRequestId = printRequestId+1) }`，
`printRequestId` 遞增確保即使同一張單也會重新觸發 [[MainActivity入口]] 的列印觀察。

> 早期確實「無新單自動列印、只剩手動 Reprint」；2026-06-04 commit ed9619c「接單即印廚房單」改為接單即印。
> `printMode == 1` 路徑仍註解掉，目前僅 mode 0 有效。手動 Reprint（見下節）與加點補印另計。

## 印表機型號設定（hardcoded）

- 紙寬：[[Prefs偏好設定]] `printerIs80mm`（預設 true）。**2026-06-25 起 `EscPosReceiptBuilder` 分隔線用它決定寬度** `receiptDashLine(is80mm)` → 80mm 48 dash / 58mm 32 dash（原寫死 30 dash，80mm 只佔 ~62% 不美觀）。⚠️ 此節點下方「列印序列」描述的是**舊 EPSON ePos SDK** 版（`Printer(22)`/`addTextSize`/`createReceiptData`），現行收據已改 **`EscPosReceiptBuilder`（ESC/POS over TCP）**，舊段落待更新
- 模型編號：`Printer(22, ...)` 硬編 22（EPSON TM 系列代碼）
- 語系：`if (prefs.language == "English") 0 else 2`

## 2026-06-04~05 列印內容調整
- **套餐主項(GType G/M)不印數量**（只印名稱）；附餐(S/R)照常。
- **加點單**：只印新增品項（2026-06-05 起**不再印**「加点 / ADD ORDER」標頭；`order.addonPrint` 仍供上游判斷印哪些品項）（見 [[加點處理]]）。
- **加料/調味第二語言**：依 itemDisplayLan，雙語『英 [中]』；資料源見 [[POS-API端點]]（JOIN 主檔）。
- **列印失敗可重印**：state.printRequestId 重觸發機制 + RetryPrintOrder + ErrorDialog『重印』鈕（含一般廚房單與加點單）。



## 2026-06-05 廚房單列印修正
- **補印加料(addition)**：廚房單組版(`EscPosReceiptBuilder`)原本只印調味(flavor)、漏了加料，現補上(套餐主項與一般品項都含 flavor+addition)。
- **加點補印移除標頭**：加點補印的收據不再印「加点 / ADD ORDER」(`order.addonPrint` 欄位仍保留供上游判斷該印哪些品項，但版面不再讀它)。
