# 設定頁

`SettingPage.kt` — 配置 App 與 POS 連線的所有參數。

## 主要欄位

| 欄位 | Prefs key | 說明 |
|------|-----------|------|
| 本地 IP | `localIp` | POS 服務 IP（拼接於所有本地 API URL） |
| KDS_ID | `kdsId` | 此裝置在 POS 端的識別碼 |
| RS No | `rsno` | 店家代號（推播 `OrdersNotify` 會帶） |
| 印表機 | `printerName` / `printerTarget` | 用 `PrinterDetecter` 掃描出可用清單 |
| 語系 | `language` | UI 語系 |
| 品項顯示語系 | `itemDisplayLan` | 訂單細項用哪個語系顯示 |
| 預設頁 | `defaultPage` | 啟動時的 Tab（目前主要用 Main） |
| 系統模式 | `mode` | 0=KDS / 1=OrderReady（見 [[系統模式切換]]） |
| 列印模式 | `printMode` | 0/1（受 `PrintRadio` 控制） |
| 準備模式 | `prepareMode` | Boolean（受 `PrepareRadio` 控制） |

## 主要互動

- `ModeRadio` → `OnModeChanged` → 切換 KDS / OrderReady（[[系統模式切換]]）
- `PrintRadio` → `onPrintModeChanged`
- `PrepareRadio` → `onPrepareModeChanged`
- 印表機 dropdown → `OnPrinterSelected(info)`
- 語系 dropdown → `OnLanguageChanged` / `OnItemDisplayLanguageChanged`

## 進入時機

從 `KdsScreen` 或 `OrderReadyScreen` 進入（兩者都有 `navigateToSetting` 入口）。
按返回時走 `navigateTo("kds")`，由 KdsScreen 的 `LaunchedEffect(modeState)` 決定是否再導向 OrderReady。

## 2026-06-04 更新：取餐牆方向設定

| 項目 | 說明 |
|------|------|
| **取餐牆方向**（新） | `OrientationRadio`（Landscape / Portrait），位於 System Mode 下方；綁 `prefs.orderReadyOrientation`(0=橫 1=直)。事件 `onOrderReadyOrientationChanged(Int)` → 寫入 prefs |
| 套用點 | `MainActivity` 的 `orderReady` route `LaunchedEffect` 依 `prefs.orderReadyOrientation` 設 `requestedOrientation`(LANDSCAPE/PORTRAIT)，取代先前的 SENSOR |
| 生效時機 | 進入/重入取餐牆頁（或重啟 App）時套用；OrderReadyScreen 依 `configuration.orientation` 連動切換每組顯示數(橫8/直6)與字級。詳見 [[OrderReady模式]] |
| 字串 | `order_ready_orientation`（en: Order Ready Orientation / zh: 取餐墙方向） |

## 2026-06-05 Server URL 預設值改為 BASE_URL（雲端）
- `DEFAULT_SERVER_URL` 從 `192.168.0.162:8099`（本地 POS）改成 `Constants.BASE_URL`（雲端後台，目前 UAT）。
- 效果：Server URL **留空**時，遠端售罄/推播會打到雲端後台，不再被導向本地 POS。
- ⚠️ **留空陷阱（已修）**：舊預設是本地 POS IP → 遠端 SetSellStatus 其實打到本地 POS 同一支端點、**會「假成功」但沒同步雲端 ProductBoutique**。改 BASE_URL 後留空＝明確打雲端。
- 切 UAT↔正式只需改 `Constants.BASE_URL` 一處，預設值自動跟著走。
- 已設過值的機器不受影響（只改 blank 回退）。見 [[POS-API端點]]、[[ISSUE-測試用預設URL待移除]]。
