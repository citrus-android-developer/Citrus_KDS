# 設定頁安全鎖

進設定頁時的驗證鎖（`SecurityDialog`）。防止非授權者誤入/亂改設定。

## 流程

- `SettingPage.kt:125,137` `securityManager` 在 `firstInstall` 時為 true → `:503` 顯示 `SecurityDialog`。
- `SecurityDialog.kt`：畫面上方顯示 **`code`（挑戰碼）**，使用者輸入驗證碼。
- `:198` 比對 `輸入 == finalCode` → 相符：`prefs.firstInstall = false` + `finish()` + 關閉；不符：紅色錯誤 icon。

## 解鎖演算法（`Constants.kt:193-206`）

```
dateStr = SimpleDateFormat("MMdd").format(今天)     // 裝置當天，如 06/17 → "0617" → 617
code    = (Math.random()*99999 + 100001).toInt()    // 100001~199999 隨機，畫面顯示
finalCode = ( reverse(code) + 525111 ) * 3 + dateStr.toInt()
```

- `reverse(code)`：字串反轉（"137487" → "784731"）。
- 魔術常數：`525111`、`×3`；日期用 `MMdd`（前導零會被 `toInt()` 去掉）。

**手算範例**（code=137487、06/17）：
`(784731 + 525111) × 3 + 617 = 1309842 × 3 + 617 = 3929526 + 617 = 3930143`

> ⚠️ **日期相依**：`finalCode` 含當天 `MMdd`，**換日就變**；換挑戰碼也變。算解鎖碼前先確認裝置日期與當下顯示的 code。

## ★DEBT★ 安全性

演算法**寫死在 client、完全可逆**——任何拿到此公式的人都能從畫面 code 推出 finalCode。屬「**防誤觸 / 防一般員工亂改**」等級，**非防惡意**。若要真安全需改伺服器端挑戰/簽章驗證（目前無此需求，標 DEBT 不標 INVARIANT）。
