# 版本更新系統

從 HTTP server 下載 APK 並安裝。手動觸發（使用者輸入版本號）。

## 觸發入口

兩個地方點擊版本號標籤都會走 `intentToUpdate()`（`MainActivity.kt:223-243`）：
- [[KdsScreen容器]] 底部 "Version.X.X" 標籤
- [[OrderReady模式]] 設定按鈕（OrderReadyScreen.kt:198）

流程：
```
intentToUpdate:
  checkSelfPermission(WRITE_EXTERNAL_STORAGE)
        │
   有權限 ─→ updateAsk.value = true → 顯示 UpdateDialog
        │
   無權限 ─→ requestPermissions(..., 888)  ★ 沒處理授權後續，需再按一次
```

## UpdateDialog（輸入版本號）

`widget/UpdateDialog.kt`：

- TextInputField 輸入版本號（KeyboardType.Number）
- 「Cancel」/「Download」按鈕
- Download 點下 → `finish(versionStr)` → 派發 `IntentToUpdateVersion(version)`

## ViewModel 處理

`CentralViewModel.kt:258-263`：

```kotlin
is Event.IntentToUpdateVersion -> {
    intentDownloadApk(event.version)
}
```

```kotlin
private fun intentDownloadApk(version: String) {
    updateJob = viewModelScope.launch {
        downloadApkUseCase.downloadApk(version, onProgressCallBack = { progress ->
            setState { copy(downloadStatus = DownloadStatus.Progress(it)) }
        }).collect {
            setState { copy(downloadStatus = it) }
        }
    }
}
```

## KtorDownloadUseCase

`presentation/usecase/KtorDownloadUseCase.kt` — 用 **Ktor 而非 Retrofit**（單獨情境，因為要 progress 回呼）。

```
URL = "http://hq.citrus.tw/apk/compass_kds_v${version}.apk"
        │
HttpClient().get(url) {
    onDownload { bytesSentTotal, contentLength ->
        progress = (bytesSentTotal * 100f / contentLength).toInt()
        onProgressCallBack(progress)
    }
}
        │
寫到 getExternalFilesDir(DOWNLOADS) / "compass_kds.apk"
        │
emit DownloadStatus.Success / Error
```

## DownloadStatus（sealed class）

```kotlin
sealed class DownloadStatus {
    object Success
    data class Error(message: UiText)
    data class Progress(progress: Int)
}
```

## 設計觀察

### 缺少的部分
- **下載完成後沒有自動安裝**：寫進 Downloads 資料夾就結束，使用者要自己去檔案管理員開 APK
- **權限授予後不會自動繼續**：`requestPermissions` 沒實作 callback，使用者授權後要再按一次版本號
- **不檢查當前版本**：使用者要輸入要下載的版本號（不會自動偵測最新版）
- **URL 路徑寫死 hq.citrus.tw**（`KtorDownloadUseCase.kt:36`），與 `Constants.DOWNLOAD_URL` 重複

### 安全
- HTTP 而非 HTTPS（路徑 `http://hq.citrus.tw/`）— APK 可能被中間人替換
- 沒有簽章驗證

## 2026-06-04 修正：點版號沒反應（Android 11+）
- **症狀**：新機點版號標籤完全沒反應、不跳 UpdateDialog。
- **根因**：`intentToUpdate` 不分版本都檢查 `WRITE_EXTERNAL_STORAGE`，但該權限在 **Android 11+(API30+) 無法授予** → checkSelfPermission 永遠 DENIED、requestPermissions 又被系統忽略不跳窗 → `updateAsk` 永遠 false。
- **關鍵事實**：下載目標 API29+ 是 `getExternalFilesDir(DOWNLOADS)`（app 專屬目錄）**本來就免權限**。
- **修法**（MainActivity.intentToUpdate）：`if (SDK_INT >= Q) { updateAsk.value = true; return }`，只有 API<29 才檢查權限。
- **注意**：版更入口只有 **KDS 頁版號**；OrderReady 頁版號是連到設定（非版更）。
- 驗證：實機點 KDS 版號 → 正常跳『版本更新』對話框 ✅。FLAG PERMISSION_NO_CALLBACK 對 API30+ 已不適用（直接略過權限）。
