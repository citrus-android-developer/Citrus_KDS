# 版本更新系統

從 HTTP server 下載 APK 並安裝。手動觸發（使用者輸入版本號）。

## 觸發入口

兩個地方點擊版本號標籤都會走 `intentToUpdate()`（`MainActivity.kt:223-243`）：
- [[KdsScreen容器]] 底部 "Version.X.X" 標籤
- [[OrderReady模式]] 設定按鈕（OrderReadyScreen.kt:198）

流程：
```
intentToUpdate:
  checkSelfPermission(WRITE_EXTERNAL_STORAGE)
        │
   有權限 ─→ updateAsk.value = true → 顯示 UpdateDialog
        │
   無權限 ─→ requestPermissions(..., 888)  ★ 沒處理授權後續，需再按一次
```

```kotlin
private fun intentDownloadApk(version: String) {
    updateJob = viewModelScope.launch {
        downloadApkUseCase.downloadApk(version, onProgressCallBack = { progress ->
            setState { copy(downloadStatus = DownloadStatus.Progress(it)) }
        }).collect {
            setState { copy(downloadStatus = it) }
        }
    }
}
```

## 2026-06-04 修正：點版號沒反應（Android 11+）
- **症狀**：新機點版號標籤完全沒反應、不跳 UpdateDialog。
- **根因**：`intentToUpdate` 不分版本都檢查 `WRITE_EXTERNAL_STORAGE`，但該權限在 **Android 11+(API30+) 無法授予** → checkSelfPermission 永遠 DENIED、requestPermissions 又被系統忽略不跳窗 → `updateAsk` 永遠 false。
- **關鍵事實**：下載目標 API29+ 是 `getExternalFilesDir(DOWNLOADS)`（app 專屬目錄）**本來就免權限**。
- **修法**（MainActivity.intentToUpdate）：`if (SDK_INT >= Q) { updateAsk.value = true; return }`，只有 API<29 才檢查權限。
- **注意**：版更入口只有 **KDS 頁版號**；OrderReady 頁版號是連到設定（非版更）。
- 驗證：實機點 KDS 版號 → 正常跳『版本更新』對話框 ✅。FLAG PERMISSION_NO_CALLBACK 對 API30+ 已不適用（直接略過權限）。


## 2026-06-17 下載完自動安裝（補完 NO_AUTO_INSTALL）

**問題**：使用者打版號後有下載，但 ① 進度對話框停在「download success」不消失、② 只下載沒安裝。根因：安裝功能從未寫完——權限(`REQUEST_INSTALL_PACKAGES`)+ FileProvider(`kds.provider` + provider_paths)plumbing 都有，但 `Effect.DownloadApkSuccess` **無人收**、Dialog 的 Success 分支**無 dismiss/安裝鈕**。

**修法**（`MainActivity.kt`）：
- setContent 內 `LaunchedEffect(Unit)` collect `homeViewModel.effect`，收到 `DownloadApkSuccess` → 派發 `OnDismissDownloadApkDialog`（關對話框）+ `installApk()`。
- `installApk()`：API26+ 先檢查 `packageManager.canRequestPackageInstalls()`，false → `unknownSourcesLauncher.launch(ACTION_MANAGE_UNKNOWN_APP_SOURCES)`（`registerForActivityResult`，返回後若已授權自動 `launchApkInstaller()`）；已授權 → 直接 `launchApkInstaller()`。
- `launchApkInstaller()`：`FileProvider.getUriForFile(this,"kds.provider",file)` → `ACTION_VIEW` + `application/vnd.android.package-archive` + `FLAG_GRANT_READ_URI_PERMISSION|FLAG_ACTIVITY_NEW_TASK` → 系統安裝器。檔案 = `getExternalFilesDir(DOWNLOADS)/compass_kds.apk`（與下載存檔同路徑）。
- UX 決策（使用者確認）：下載完**自動**跳安裝器（非加按鈕）；未知來源未授權**導設定頁**。
- 驗證：compile + 實機 SM-X710 install -r 啟動無 crash。⚠️ 該機裝 debug、下載的是 CICD release 簽章 APK → 系統安裝器會因**簽章不符**拒裝，只能驗到「安裝器跳出」；簽章一致的 production 機才能完成安裝。見 [[Verification/2026-06-17_更版自動安裝]]。
- **同批修 UpdateDialog 卡前景**（既有 bug）：`MainActivity.kt` 的 `UpdateDialog.finish`（點 Download）原本只派發 `IntentToUpdateVersion`、沒設 `updateAsk.value=false` → 版本輸入框點下載後不關、卡前景。補 `updateAsk.value = false`。production 端到端(1.1.8→1.1.9 OTA 自動安裝)經使用者實測通過。

## 2026-06-26 安裝權限「下載前」先索取（修要按第二次下載）

**問題**：點版號下載，第一次跑完**進度條直接消失、卻沒跳安裝器**，要再按第二次下載才跳「版本更新」(系統安裝器)。

**根因**：安裝被 `canRequestPackageInstalls()`（安裝未知應用程式權限）這道閘擋住，而它是 #1/#2 間唯一會變的狀態（APK 兩次相同）。下載完 `Effect.DownloadApkSuccess` 一面**關進度條**（=進度條消失）一面 `installApk()`；首次權限未授 → `installApk()` 走 `unknownSourcesLauncher` 導設定頁後 `return`、**不安裝**，而授權返回的 auto-resume `canRequestPackageInstalls()` 常回**未刷新的舊值(false)**（Android 已知時序問題）或使用者直接返回 → 沒續裝。第二次下載時權限已 settle → 直接跳安裝器。

**修法**（`MainActivity.kt`，純前端）：把權限閘**提前到下載前**。
- `updateAsk` 由 setContent 內 `remember` 提到 **Activity 層級 `mutableStateOf`**（讓權限 launcher 回呼也能開輸入框、跨重組存活）。
- 新增 `installPermissionLauncher`：`intentToUpdate()`（API29+）若 `!canRequestPackageInstalls()` 先導 `ACTION_MANAGE_UNKNOWN_APP_SOURCES`，**返回即開 `UpdateDialog`**（不論授權與否；保底仍有 `installApk()` 導頁）。已授權則直接開輸入框。
- 結果：使用者按 Download 下載完成時權限**早已就緒**，`installApk()` 直接 `launchApkInstaller()` 跳安裝器，**不再需要按第二次**。
- `installApk()` / `unknownSourcesLauncher`（下載後導頁）保留為保底，未刪。
- 驗證：compile OK；待實機驗收（1.1.18）。
