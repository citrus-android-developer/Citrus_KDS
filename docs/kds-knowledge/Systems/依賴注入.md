# 依賴注入（Hilt）

Hilt + Dagger，所有元件單例。

## Application 入口

`di/MyApplication.kt`：

```kotlin
@HiltAndroidApp
class MyApplication : Application() {
    @Inject lateinit var mPrefs: Prefs
    
    override fun onCreate() {
        super.onCreate()
        prefs = mPrefs                                  // 寫入 companion 單例
        Timber.plant(MultiTagTree())
    }
    companion object { var prefs: Prefs? = null }
}

val prefs: Prefs by lazy { MyApplication.prefs!! }      // 全域變數
```

> 全域 `prefs` 是頂層變數，整個 codebase `import com.citrus.citruskds.di.prefs` 直接拿 — 任何地方都能改設定，**沒有封裝邊界**。改了哪些值靠語意推論。

## AppModule（SingletonComponent）

`di/AppModule.kt`：

| Provides | 用途 |
|----------|------|
| `okHttpClient()` | OkHttp，含 ApiKey header + HttpLoggingInterceptor (Headers+Body level) |
| `provideRetrofit(okHttp)` | Retrofit 用 `Constants.BASE_URL` 為 baseUrl + Moshi converter |
| `provideApiService(retrofit)` | `retrofit.create(ApiService::class.java)` |
| `providePref(application)` | [[Prefs偏好設定]] 單例 |
| `providePrintUtil(app, printerDetecter)` | [[列印系統]] PrintUtil 單例 |
| `provideSharedPreference(application)` | SharedPreferences 直接 instance（給 Prefs 用，但 Prefs 自己又呼叫 getSharedPreferences — **重複**） |

注意：`PrinterDetecter` 是 `@Singleton class @Inject constructor()` 自帶 binding，沒在 AppModule。

## OkHttp ApiKey 攔截器

`AppModule.kt:42-49`：

```kotlin
.addInterceptor {
    val request = it.request().newBuilder()
        .addHeader("ApiKey", "CitrusCompassKDS".sha3_256())
        .addHeader("Content-Type", "application/json")
        .build()
    it.proceed(request)
}
```

`sha3_256()` 在 `Constants.kt:178-181` 是**寫死回傳常數**：

```kotlin
fun String.sha3_256(): String {
    return "5EC8433D25C759DD6BB965090F6835C77BB569CE86F3713B2D364E642F693280"
}
```

所有請求帶同一個 ApiKey，沒有真的 hash — 屬於**識別而非認證**，內網信任前提。原始 SHA3 實作在上方註解中。

## Timber Logging

`di/MultiTagTree.kt`：自訂 tag 格式為 `[functionName][className]`，方便 Logcat 過濾。Debug 用，無 release 配置。

## 設計觀察

- **頂層全域 prefs**：`val prefs: Prefs by lazy { MyApplication.prefs!! }` — 用法方便但測試地獄，無法替換 mock
- **重複的 SharedPreferences provide**：AppModule 同時提供 `Prefs` 和 `SharedPreferences`，Prefs 內部又自己 `getSharedPreferences`，三個地方拿同樣的 instance
- **HttpLogging 留在 release**：兩個 BODY-level 攔截器堆疊，會在 logcat 印出所有訊息（含可能的敏感資訊）
- **ApiKey 寫死**：認證等同無
- **Repository 由 ApiRepositoryImpl 自帶 binding**：沒在 AppModule 顯式宣告（透過 `@Singleton @Inject constructor`）
