# 震動器號碼顯示修復驗證

## 現象
KDS 主卡片單號旁本應顯示震動器號碼 ` (123)`，實機從未出現。

## 根因（實際 API 證實）
打實際 API `POST http://192.168.0.162:8099/KDS/OrdersList`（ApiKey=SHA256("CitrusCompassKDS") 大寫 hex）回傳：
```json
{"OrderNo":"W012026061700008","Status":"W","OrderNote":"123\\","Detail":[...]}
```
JSON key 是 **`OrderNote`**，但前端 `Order.note` 標 `@Json(name = "Note")`（`Order.kt:17`）→ Moshi 對不到 → `note` 永遠 null → `OrderItem.kt:109` 的 pager 永遠空。
git 證實此 bug 自功能引入 commit `ab02278`（在主卡單號旁顯示震動機號碼）起**從未生效**，非迴歸。

## 修正
1. `Order.kt`：`@Json(name = "Note")` → `@Json(name = "OrderNote")`。
2. `Order.kt`：新增衍生屬性 `Order.buzzerNo`（去尾綴 `\`、trim、空回 null）。
3. `OrderItem.kt:109`：`val pager = order.buzzerNo?.let { " ($it)" } ?: ""` + import。

## 測試（TDD，綁定合約）
`app/src/test/java/com/citrus/citruskds/OrderBuzzerTest.kt`（Moshi 反序列化）：
| 案例 | 預期 | 結果 |
|------|------|------|
| `"OrderNote":"123\\"` → note / buzzerNo | `"123\\"` / `"123"` | ✅ |
| `"OrderNote":"\\"`（只反斜線）→ buzzerNo | null | ✅ |
| `"OrderNote":""` → buzzerNo | null | ✅ |
| 缺 OrderNote 欄位 → buzzerNo | null | ✅ |

- 先紅（改前第 1 案失敗，note=null）→ 改 `@Json` 後綠：`./gradlew :app:testDebugUnitTest --tests "com.citrus.citruskds.OrderBuzzerTest"` BUILD SUCCESSFUL（4/4）。
- `:app:compileDebugKotlin` 通過。

## 裝置端確認（2026-06-17 已驗）
Samsung SM-X710 裝 1.1.6(vc17, debug 含修正)，Main 頁實機截圖確認：
- `W01-00008 (123)`、`W01-00001 (254)`、`W01-00003 (545)`、`W01-00004 (744)` → 震動器號碼正常顯示。
- `W01-00005`(OrderNote 僅 `\`)、`T02-00003/00005`、`P04-00001`(OrderNote 空) → 正確不顯示括號（去尾綴 + 空值判斷成立）。
> 安裝註記：裝置原 1.1.5 為**另一把 keystore**(非 repo app/kds_key.jks)所簽，install -r 簽章衝突 → 改 uninstall 後裝；先 run-as 備份/還原 sharedPref.xml 保留連線設定(localIp 192.168.0.162:8099 / KDS_ID 01 / printer 192.168.0.66)。
