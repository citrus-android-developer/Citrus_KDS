# 取餐牆召回刷新時間→變紅 端到端驗證

## 結果：PASS（D3_PRO 實機）

## 場景
召回已在取餐牆(O)的單 → 後端 Finishtime=GETDATE() → 依 Finishtime DESC 重排往前 → App 端「超車偵測」判定變紅。

## 測試項目
1. D3 切 OrderReady 模式，牆上 Citrus 群組 = 0026,0027,0017（全白，無紅）。✓
2. API 召回最舊 P082026060500017（KDS_ID=01,Status=O,FromStatus=O,F）→ 跳到最前，但**第一次沒變紅**。✗→修。
3. **發現 bug**：偵測用 OrderName 當 key，但有 3 個群組同名 "Citrus"(PrintGroup 01/03/04)→ associate 覆蓋 → prev 變空 → 抓不到超車。改用**群組索引**當 key。
4. 重測召回 P08-0027 → **跳到最前 + 紅底高亮**(其餘白字)+ 觸發叮咚。✓
5. 後端 SetStatus WHERE 有 `PrintGroup=@KDS_ID` → 召回 KDS_ID 必須是品項 PrintGroup（此例 01）。✓
