# 加點處理（追加品項）

設計文件：`docs/plans/2026-06-04-加點處理-design.md`

## 需求
已在 KDS 的單被加點（新品項以 j 進來、原品項已 W/O）時：卡片顯示回新單 → 接單(手動或自動接單)只把新 j→W → 只印新增品項並標「加點」→ 同單再加點重複。

## 偵測（無狀態）
- 加點 = 同單有 j(未接) 且有 W/O(已接)；全新單 = 全 j。
- 卡片派生狀態：有 j→新單 / 否則 W→製作中 / 否則 O→待取。
- itemStatus 為 null（後端未部署）時全回退整單 status，無回歸。

## 已實作
### 後端 Compass_KDS（**待部署到 192.168.0.162**）
- `OrdersDetail.ItemStatus` 帶出 per-item 狀態（main 已 SELECT，served/recall 既有）。
- `SetStatus` 加 `FromStatus`（逗號分隔來源狀態），`UPDATE ... AND ItemStatus IN @FromStatuses`，只搬對應品項；空=整單(向後相容)。**修正混合狀態 bug**（原本整單一律改）。清掉舊死碼。

### 前端 CitrusKDS（已裝 D3+SM）
- `Detail.itemStatus`、`SetOrderStatusRequest.fromStatus`、`Order.addonPrint`。
- `Order.displayStatus()/isAddon/addonItems`、`Detail.isPending`。
- 卡片按鈕(MainFeatureBtn)、底色(OrderItem) 改用 displayStatus()。
- 事件帶 fromStatus：接單 j,J→W；完成 W→O；取餐 O→F；召回 O,F→O。
- 接單列印分流：加點→只印 addonItems + EscPos 標「加点/ADD-ON」；全新單→整張。
- 自動接單：候選改 displayStatus()==J，isAddon 可越過 baseline/InFlight 去重 → 支援再加點。

## 2026-06-04 列印失敗重印（方案 A）
加點單列印失敗時，因狀態已升 W、加點偵測消失 → 原本無法重印。修法：
- State 新增 `printRequestId`，列印觸發改 key 在此（MainActivity LaunchedEffect），確保「重印同一張(含加點子集)」也會重觸發。
- 所有列印改走 `requestPrint(order)` helper。
- 新增事件 `RetryPrintOrder`：重印 `currentState.printOrder` 的上一張快照（含加點子集與 addonPrint 標記），不受狀態已升級影響。
- `ErrorDialog` 加選配 `confirmText/onConfirm`；列印失敗對話框顯示「重印/Reprint」鈕 → 派發 RetryPrintOrder。
- 失敗→重印→再失敗 會再次跳對話框，可重複重印。
