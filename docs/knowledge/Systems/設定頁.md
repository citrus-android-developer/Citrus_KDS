# 設定頁

`SettingPage.kt` — 配置 App 與 POS 連線的所有參數。

## 主要欄位

| 欄位 | Prefs key | 說明 |
|------|-----------|------|
| 本地 IP | `localIp` | POS 服務 IP（拼接於所有本地 API URL） |
| KDS_ID | `kdsId` | 此裝置在 POS 端的識別碼 |
| RS No | `rsno` | 店家代號（推播 `OrdersNotify` 會帶） |
| 印表機 | `printerName` / `printerTarget` | 用 `PrinterDetecter` 掃描出可用清單 |
| 語系 | `language` | UI 語系 |
| 品項顯示語系 | `itemDisplayLan` | 訂單細項用哪個語系顯示 |
| 預設頁 | `defaultPage` | 啟動時的 Tab（目前主要用 Main） |
| 系統模式 | `mode` | 0=KDS / 1=OrderReady（見 [[系統模式切換]]） |
| 列印模式 | `printMode` | 0/1（受 `PrintRadio` 控制） |
| 準備模式 | `prepareMode` | Boolean（受 `PrepareRadio` 控制） |

## 主要互動

- `ModeRadio` → `OnModeChanged` → 切換 KDS / OrderReady（[[系統模式切換]]）
- `PrintRadio` → `onPrintModeChanged`
- `PrepareRadio` → `onPrepareModeChanged`
- 印表機 dropdown → `OnPrinterSelected(info)`
- 語系 dropdown → `OnLanguageChanged` / `OnItemDisplayLanguageChanged`

## 進入時機

從 `KdsScreen` 或 `OrderReadyScreen` 進入（兩者都有 `navigateToSetting` 入口）。
按返回時走 `navigateTo("kds")`，由 KdsScreen 的 `LaunchedEffect(modeState)` 決定是否再導向 OrderReady。
